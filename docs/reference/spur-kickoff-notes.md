### Content Telemetry Standards - Core Requirements

- Primary focus: establishing right to telemetry from AI systems at inference time
- Two key event types needed:
  1. Retrieval events - content pulled into AI system
  2. Citation events - content actually shown to users
- Simple building blocks approach vs. complex granular detail initially
- Authentication layer for bot identification (leveraging existing solutions like Cloudflare, Tolbit)

### Technical Architecture Overview

- Content licensing happens human-to-human for big deals initially
- Machine-readable license terms stored externally (potential blockchain use case)
- Telemetry reporting via simple web protocols (similar to pixel tracking)
- Unique content identifiers per AI system for auditability
- Real-time citation reporting preferred over batch processing

### Implementation Challenges & Solutions

- Volume concern: agents need to send telemetry to thousands of individual publishers
- Proposed centralized resolver/concentrator service to distribute telemetry
- Foundation models responsible for downstream API customer compliance
- License terms embedded in content metadata or external lookup system
